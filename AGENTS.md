# AGENTS.md

## 仓库用途

本仓库是一个多 Agent 智慧旅游规划系统的研究原型，用于学术实验。

## 工作规则

- 使用兼容 Python 3.11 的代码。
- 除非任务明确要求，不要改变用户可见行为。
- 除非明确要求，不要改变意图路由、Agent 选择、Prompt、工具调用顺序或 API 响应格式。
- 未经明确批准，不要新增生产依赖。
- 不要提交 API Key、token、密码、`.env` 内容或任何私有数据。
- tracing 和实验功能默认保持关闭。
- 每次改变行为时，都要新增或更新测试。
- 先运行最小相关测试集，再运行 `pytest -q`。
- 汇报所有执行过的命令及其结果。
- 不要直接合并到 `master`。
